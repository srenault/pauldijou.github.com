Intertitres (intership)

March - September 2010 : 6 months
Project type
Internship
Description
During my internship at CELAD, I had to develop from scratch a new administration application to manage and monitor other commercial websites. Based on Java EE, Seam 2 and JSF, this project was a perfect opportunity to learn lots of things since I've done everything from the specification redaction to developing and testing the application.
Technologies
Java, Java EE, Spring, Seam 2, JPA, JSF, RichFaces
Tags
internship java javaee spring seam jpa jsf richfaces

The actors

Intertitres is a subsidiary of Natixis (a French bank) specialized in selling securities. They have several websites in order to do that depending on the type of the securities and the size of the clients (bigger clients have better applications with more features).

Those website are mostly developed by CELAD, the IT firm in which I was doing my internship. I was working in the headquarters in Balma, near Toulouse (France) inside the team dedicated to Intertitres which was composed of 6 members at that time.

The technologies

Even if Natixis delegates most of development to IT firms, they have their own core development department in order to give global directions and roadmap to everyone. Another task on their todo list is the development and the maintenance of Sphinx.

Sphinx is an internal framework based on Java, Java EE, Spring, Seam 2 and JSF (with RichFaces). Its purpose is to give an abstract layer over all those technologies to ease development and increase productivity but also give a common structure to all Natixis’ applications. At that time, it was quite a great stack to work with.

The project

All the previous websites we’ve been talking about are connected to a central database and manipulate the same tables. With each website growing, the necessity of having a cross-application administration console in order to see and manage all the data without any relation to each website appears.
